# 旅行推荐助手测试手册

## 阶段一：信息补全测试

### T01 — 信息完整，直接进入推荐

输入：
```
7月去云南，2大1小，预算5000，爱吃辣
```

预期：
- 阶段一：信息完整，不进追问循环
- 阶段二：调 query_weather、query_attractions、query_restaurants
- 输出：符合 ItineraryOutput 格式的完整行程
- summary：写入且包含目的地+时间+人数+关键数据

---

### T02 — 缺目的地

输入：
```
7月出去玩
```

预期：
- LLM 提取：travelDates="7月"，destination=null
- 代码检查 destination 缺失
- Agent 追问："你想去哪里？"
- 用户补充目的地后才进入阶段二

---

### T03 — 缺时间

输入：
```
去云南
```

预期：
- LLM 提取：destination="云南"，travelDates=null
- 代码检查 travelDates 缺失
- Agent 追问："你打算什么时候去？"
- 用户补充时间后才进入阶段二

---

### T04 — 只缺选填（预算/人数/偏好）

输入：
```
7月去云南
```

预期：
- LLM 提取：destination="云南"，travelDates="7月"，其他选填为 null
- 代码检查必填都有，直接进入阶段二
- LLM 可在推荐时自行询问题问预算和人数，也可以不问

---

## 阶段二：推荐主循环测试

### T05 — 正常推荐流程

输入：
```
7月去云南，2大1小
```

预期：
- LLM 思考需要查数据
- query_weather("云南", "7月") → 返回天气
- query_attractions("云南", "亲子") → 返回景点
- query_restaurants("云南", "亲子") → 返回餐厅
- LLM 编排行程 → 输出 ItineraryOutput
- 写入 summary，清空旧消息

---

### T06 — 天气查询超时/失败

输入：
```
7月去云南
```
（模拟 query_weather API 超时或报错）

预期：
- query_weather 返回错误
- 错误信息传给 LLM："天气查询暂时不可用，按无天气数据推荐"
- LLM 继续推荐但不包含天气建议

---

### T07 — 景点查不到

输入：
```
去南极
```

预期：
- query_attractions("南极", "") 返回空结果或 LLM 无此知识
- LLM 告知用户：暂无可推荐的景点，建议换个目的地
- 不强行编造景点

---

## 阶段三：后续交互测试

### T08 — 局部修改：换酒店

前置：已完成一次推荐

输入：
```
换个便宜的酒店
```

预期：
- LLM 判断为局部修改
- 只调 query_hotels，不重新查天气/景点/餐厅
- 更新行程中的酒店部分，其他部分不变

---

### T09 — 局部修改：改预算

前置：已完成一次推荐

输入：
```
预算调到3000
```

预期：
- LLM 判断为局部修改
- 不调任何工具，直接在现有行程基础上重新估算费用
- 更新预算部分

---

### T10 — 全局修改：换目的地

前置：已完成一次推荐

输入：
```
换成大理
```

预期：
- LLM 判断为全局修改
- 清空之前查到的天气/景点/餐厅/酒店数据
- 回到阶段二，从头查起

---

### T11 — 全局修改：换日期

前置：已完成一次推荐

输入：
```
改成12月去
```

预期：
- LLM 判断为全局修改
- 清空之前查到的天气数据（日期变了，旧天气无效）
- 回到阶段二，从头查起

---

## 权限控制测试

### T12 — 只读模式保存

启动：`python cli/main.py`（不带 --writable）

前置：已完成一次推荐

输入：
```
保存
```

预期：
- Agent 调用 save_itinerary
- save_itinerary 检查到未启用 --writable
- 返回报错："当前为只读模式，请用 --writable 重新启动"

---

### T13 — 可写模式保存

启动：`python cli/main.py --writable`

前置：已完成一次推荐

输入：
```
保存
```

预期：
- Agent 调用 save_itinerary
- save_itinerary 检查到已启用 --writable
- 写入 saved/ 目录，文件名根据目的地+时间自动生成
- 返回："已保存到 saved/云南亲子7日游.md"

---

## 边缘情况测试

### T14 — 输入乱码

输入：
```
asdfghjkl
```

预期：
- LLM 无法理解输入
- 追问："抱歉，我没理解你的意思。请告诉我你想去哪里旅行？"

---

### T15 — 空输入

输入：
```
（直接回车）
```

预期：
- CLI 层不做处理，重新显示提示符
- 或提示："请输入旅行需求"

---

### T16 — 超长对话（多轮修改）

前置：已完成一次推荐

操作：
```
> 换成大理      （第1轮）
> 改成12月      （第2轮）
> 加一天        （第3轮）
> 预算调低      （第4轮）
> 换个酒店      （第5轮）
> 加一个景点    （第6轮）
> 再改回大理    （第7轮）
... 继续多次
```

预期：
- 每轮推荐完成后 summary 正常更新
- LLM 上下文始终为 [summary] + [当前轮]，不会撑爆
- 推荐质量不因轮数多而下降

---

### T17 — 中途退出

操作：推荐过程中按 Ctrl+C

预期：
- 程序优雅退出，不报错堆栈
- 下次启动是新会话，不保留任何数据

---

## 测试环境准备

```
1. 安装依赖：pip install openai
2. 设置环境变量：
   export OPENAI_API_KEY=xxx
   export AMAP_KEY=xxx
3. 启动：
   python cli/main.py              # 只读模式
   python cli/main.py --writable   # 可写模式
4. 测试目录：
   saved/    保存的行程文件（--writable 模式）
```
